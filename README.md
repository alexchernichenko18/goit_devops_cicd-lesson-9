# Як працює весь пайплайн

Push у репозиторій `example-app` > Jenkins pipeline стартує
Jenkins збирає Docker-образ
Jenkins пушить образ у ECR (з тегом `latest`)
Jenkins оновлює тег у `values.yaml` Helm-чарту (в репозиторії `infra`). Цей крок, насправді, не стільки потрібен, так як тег завжди буде latest, я його не міняю в `example-app`.
Jenkins пушить це в GitHub
Argo CD бачить зміну у Helm-чарті > оновлює застосунок у кластері eks
Новий образ із ECR підтягується і деплоїться

# Нотатки

## Як застосувати Terraform

Як в попередніх дз, спочатку застосувати `terraform init` з закоменченим бекендом, щоб підвантажились S3 бакет та DynamoDB. `terraform apply`.

Після цього розкоментити все інше, `terraform init -reconfigure`, `terraform apply`. В мене не з першої спроби завантажилось EBS CSI Driver, можна спробувати декілька разів.

## Як перевірити Jenkins job

На UI в Jenkins побачимо нашу створену джобу `seed-job`, яка знаходиться в `values.yaml`. Натиснемо Build Now, наш образ підвантажиться в ECR. Резільтати деплою можна побачити в `Builds`.

## Як побачити результат в Argo CD

В інтерфейсі ArgoCD ми можемо бачити, що оновлений Deployment створює новий pod `django-django-5f94769698-hnv5p`. Успішність деплою контролюємо по зелених сердечках. Але те саме можна робити з командою `kubectl get pod`, на якій можна глянути, чи pod `ready`, і якщо ні, то чекнути через `kubectl logs django-django-6c5b8c7895-hnv5p`.
